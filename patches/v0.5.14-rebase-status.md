**NEW patch (caught by validation):**
- **056 gdn-conv-state-dtype-cast** — v0.5.14's GatedDeltaNet backend grew a `has_mamba_track_mask`
  path (the new mamba-track / `extra_buffer` radix-cache strategy) that `index_put`s the fp16
  conv source into the **bf16** conv-state cache: `gdn_backend.py:438
  conv_states[...mask_indices] = mixed_qkv_to_track` → `RuntimeError: Index put requires the
  source and destination dtypes match, got BFloat16 (dest) ... Half (source)`. Crashes qwen36
  (and any Qwen3.5/3.6 DeltaNet preset, which launch `--dtype float16`) on the first decode forward,
  right after the 267 s decode-CUDA-graph capture. Fix: cast the source to `conv_states.dtype` at
  the write (no-op when matched) — the torch-level sibling of patch 003's kernel-side conv_state
  cast (003's own comment notes "conv_state may be bf16 while activations are fp16 under AWQ").
  Not present on v0.5.13 (the mamba-track write path is new). **This is the one functional break the
  v0.5.13→v0.5.14 bump introduced for our fleet** — everything else was context drift.
